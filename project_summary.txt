===================================================
      ВЫЖИМКА ПРОЕКТА: "CYBER TRAINER"
===================================================

ДЛЯ ИСПОЛЬЗОВАНИЯ В ДИПЛОМНОЙ РАБОТЕ

1. ОПИСАНИЕ И АРХИТЕКТУРА
Проект "Cyber Trainer" представляет собой комплексную образовательную платформу, состоящую из трех основных модулей:
- Backend (Серверная часть)
- Frontend (Веб-платформа)
- Mobile (Мобильное приложение для студентов)

Архитектурный паттерн: Клиент-серверная архитектура с REST API.

2. ТЕХНОЛОГИЧЕСКИЙ СТЕК
- Backend: Python 3.12+, FastAPI, SQLModel (SQLAlchemy), Pydantic, JWT (авторизация).
- База данных: PostgreSQL 18.
- Веб-Frontend: React 18, Vite, TypeScript, TailwindCSS, Lucide-React.
- Мобильное приложение: React Native, Expo, TypeScript, React Navigation.

3. СТРУКТУРА БАЗЫ ДАННЫХ (Ключевые таблицы)
- users: Хранение пользователей (id, username, email, hashed_password, role). Роли: admin, teacher, student.
- courses: Учебные материалы. Типы: lesson (лекции) и exam (экзамены). Содержит time_limit и дедлайны.
- course_access: Связь Many-to-Many для назначения курсов студентам.
- questions: Вопросы для тестирования (привязаны к курсам через slug/topic). Поддерживаются типы "выбор ответа" и "свободный ввод".
- results: Результаты прохождения экзаменов студентами (связь с users и courses).
- knowledge_category / knowledge_article: Раздел "База Знаний" (категоризированные справочные материалы).

4. КЛЮЧЕВЫЕ ENDPOINTS (REST API)

[Аутентификация и пользователи]
- POST /api/auth/login : Выдача JWT токена при совпадении логина/пароля.
- POST /api/auth/register : Регистрация нового студента.
- GET /api/users/me : Получение информации о текущем пользователе.
- POST /api/student/update_profile : Изменение данных профиля студента.

[Учебные материалы (Студенты)]
- GET /api/courses : Получение списка лекций и экзаменов (с учетом прав доступа).
- GET /api/questions?topic={slug} : Получение списка вопросов для выбранного курса.
- POST /api/check_answer : Пошаговая проверка ответа студента на вопрос (на бэкенде).
- POST /api/results : Фиксация итогового балла после завершения тестирования.
- POST /api/mark-lecture-viewed : Отметка об успешном прочтении лекции студентом.

[База знаний]
- GET /api/knowledge/categories : Запрос всех категорий.
- GET /api/knowledge/articles : Запрос всех доступных статей.
- GET /api/knowledge/articles/{id} : Чтение конкретной статьи.

[Администрирование и преподаватели]
- GET/POST/PUT /api/admin/users : CRUD операции с пользователями.
- GET/POST/PUT /api/admin/courses : Управление учебными материалами.
- GET/POST/PUT /api/admin/questions : Добавление и изменение тестовых вопросов.
- GET /api/leaderboard : Формирование рейтинга студентов на основе их баллов.

5. КЛЮЧЕВЫЕ МЕХАНИЗМЫ ("МОЗГИ" ПРОЕКТА)
- Ролевая модель доступа: Доступ к веб-панели управления ограничен для учителей и администраторов. Мобильное приложение зарезервировано строго для роли "student".
- Защита тестирования: Мобильное приложение отправляет каждый ответ на сервер `/check_answer`, где бэкенд сверяет его с правильным. Правильные ответы не передаются на устройство до момента проверки, что исключает возможность их извлечения из кода.
- Таймер экзаменов: В приложении внедрен счетчик обратного отсчета, который сверяется с time_limit из БД. При достижении нуля тест принудительно завершается и сохраняет результат.
- Система трофеев: Геймификация. Локальное отслеживание достижений ("Первый шаг", "Идеальный балл" и т.д.) через защищенное хранилище (SecureStore / LocalStorage).
- Динамический рейтинг (Leaderboard): Сервер агрегирует результаты всех экзаменов и выстраивает топ-лидеров, мотивируя студентов к соревнованию.
